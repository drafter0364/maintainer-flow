# Roadmap

Maintainer Flow should grow by solving concrete maintainer problems before adding broad automation.

## Near term

- Add fixtures from real open-source review scenarios, with sensitive data removed.
- Improve release checks for changelog coverage, tag consistency, and provenance notes.
- Add a dry-run action mode that writes summaries without commenting.
- Publish a first tagged release after the action has been tested on this repository.

## Later

- Support ecosystem-specific checks for npm, Python, Go, and Rust projects.
- Add configurable policy files for repositories that want stricter review rules.
- Provide optional SARIF output for security and supply-chain findings.
- Evaluate Marketplace publishing after a stable release and clear screenshots are available.
